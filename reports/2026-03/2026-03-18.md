- **Aim**: Extend standard deep‑research tools by inferring a user’s research‑interest profile, generating personalized actions for each query, and producing a multi‑section report that follows those actions

#### Abstract
> Deep Research (DR) tools (e.g. OpenAI DR) help researchers cope with ballooning publishing counts. Such tools can synthesize scientific papers to answer researchers' queries, but lack understanding of their users. We change that in MyScholarQA (MySQA), a personalized DR tool that: 1) infers a profile of a user's research interests; 2) proposes personalized actions for a user's input query; and 3) writes a multi-section report for the query that follows user-approved actions. We first test MySQA with NLP's standard protocol: we design a benchmark of synthetic users and LLM judges, where MySQA beats baselines in citation metrics and personalized action-following. However, we suspect this process does not cover all aspects of personalized DR users value, so we interview users in an online version of MySQA to unmask them. In all, we argue for a pillar of personalization that easy-to-use LLM judges can lead NLP to overlook: real progress in personalization is only possible with real users.

#### T.A.R.G.E.T. Summary (from Full Text)
- **Topic**: IndexRAG: Offline Cross‑Document Reasoning for Retrieval‑Augmented Generation
- **Aim**: Enable cross‑document reasoning in RAG with a single retrieval pass and a single LLM call by pre‑computing reasoning links offline.
- **Rationale**: Existing RAG variants either build explicit knowledge graphs requiring online traversal or iteratively refine queries, both incurring high latency, multiple LLM calls, and often need fine‑tuned embeddings. Cross‑document reasoning patterns are largely query‑independent; thus, the logical links between documents can be pre‑computed during indexing, shifting the burden from inference to index construction.
- **Ground**: Offline pipeline: 1) Extract Atomic Knowledge Units (AKUs) and entities per document via LLM prompts. 2) Identify bridge entities with moderate document frequency. 3) Generate bridging facts that explicitly connect evidence across documents using LLM prompts on fragments. 4) Store AKUs and bridging facts in a unified dense vector index (FAISS) using a standard embedding model. Contributions include bridging facts as retrievable units, a training‑free framework, incremental updates, and compatibility with iterative methods.
- **Experiment**: Datasets: HotpotQA, 2Wiki‑MultiHopQA, MuSiQue (1,000 validation queries each). Metrics: Exact Match, Accuracy, F1 (after answer normalization); efficiency: retrieval latency, LLM calls. Results: IndexRAG achieves EM/Acc/F1 of 54.1/59.3/68.9 on HotpotQA, 44.8/50.3/51.7 on 2Wiki‑MultiHopQA, and 22.4/24.4/34.4 on MuSiQue, outperforming all single‑LLM‑call baselines by up to +4.6% F1 on average.
- **Takeaway**: IndexRAG demonstrates that pre‑computing cross‑document reasoning links as bridging facts enables efficient, single‑pass retrieval and generation, achieving state‑of‑the‑art performance among single‑LLM‑call methods while eliminating online traversal and multiple LLM calls. Its training‑free, incremental design makes it practical for real‑world deployment and combinable with iterative retrieval strategies.

#### Abstract
> Multi-hop question answering (QA) requires reasoning across multiple documents, yet existing retrieval-augmented generation (RAG) approaches address this either through graph-based methods requiring additional online processing or iterative multi-step reasoning. We present IndexRAG, a novel approach that shifts cross-document reasoning from online inference to offline indexing. IndexRAG identifies bridge entities shared across documents and generates bridging facts as independently retrievable units, requiring no additional training or fine-tuning. Experiments on three widely-used multi-hop QA benchmarks (HotpotQA, 2WikiMultiHopQA, MuSiQue) show that IndexRAG improves F1 over Naive RAG by 4.6 points on average, while requiring only single-pass retrieval and a single LLM call at inference time. When combined with IRCoT, IndexRAG outperforms all graph-based baselines on average, including HippoRAG and FastGraphRAG, while relying solely on flat retrieval. Our code will be released upon acceptance.

#### Abstract
